  4b. Limitations (~200 words)

  Write one paragraph per limitation. Keep each to 2–3 sentences.

  Limitation 1 — Single dataset generalisation:
  - The model was trained and tested exclusively on one Kaggle dataset (masoudnickparvar). All images come from a single source with a consistent format.
  - Write that MRI scans from different hospitals, scanner manufacturers, or acquisition protocols may look different enough that the model's performance
  would degrade. This matters significantly for any clinical deployment scenario.

  Limitation 2 — 2D slices lose volumetric context:
  - MRI data is inherently three-dimensional — a tumour exists across multiple consecutive slices. This project processed individual 2D slices in isolation.
  - Write that 3D volumetric models (such as 3D CNNs) would be able to exploit spatial relationships between slices, potentially improving both accuracy and
  localisation.

  Limitation 3 — Interpretability was partial:
  - Grad-CAM heatmaps were generated for a small number of example images. While these offer a qualitative view of model attention, they were not validated
  systematically.
  - Write that genuine clinical interpretability would require applying Grad-CAM across all misclassified cases and having a qualified radiologist verify
  whether the highlighted regions correspond to actual tumour locations annotated on radiology reports.

  Limitation 4 — Mixed MRI acquisition types:
  - The dataset mixes different MRI scan types (T1, T2, FLAIR) without labelling which type each image is. Different acquisition types emphasise different
  tissue properties.
  - A model trained on mixed types may be less precise when deployed in a setting that uses only one acquisition type.

  Limitation 5 — Meningioma remained the hardest class:
  - Even in the best model (EfficientNetB0 Exp A), meningioma had the lowest F1 score (0.8822) despite having exactly the same number of training images as
  other classes.
  - Write that addressing this would likely require targeted strategies such as hard example mining or class-specific augmentation, rather than simply
  collecting more data.

  4c. Future Work (~150 words)

  Write each suggestion as 2 sentences: what would be done, and why it would help.

  Suggestion 1 — Systematic Grad-CAM validation:
  - Apply Grad-CAM to every misclassified case in the test set, not just selected examples. Have clinical experts verify whether heatmap activation regions
  align with tumour boundaries documented in radiology reports — this would move interpretability from illustrative to clinically meaningful.

  Suggestion 2 — Ensemble modelling:
  - Combine predictions from EfficientNetB0, ConvNeXtSmall, and the improved custom CNN using
  soft voting or stacking. Individual model families make different types of errors, and
  ensembling tends to reduce overall error by exploiting architectural diversity.

  Suggestion 3 — 3D volumetric CNN:
  - Replace 2D slice processing with a 3D CNN that takes the full MRI volume as input. This would allow the model to capture inter-slice spatial
  relationships, which are particularly relevant for tumour boundary identification.

  Suggestion 4 — Multi-site dataset:
  - Collect and train on data from multiple hospitals and scanner types. Performance on unseen scanner protocols is the critical test for clinical deployment
   readiness.

  Suggestion 5 — Targeted meningioma improvement:
  - Apply hard example mining specifically to meningioma cases — during training, upweight the examples the model consistently misclassifies. This could
  reduce the persistent meningioma error pattern without requiring additional data collection.
