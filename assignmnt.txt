1. Introduction
1.1 Background
The use of Artificial Intelligence in medical imaging is the application of machine learning to assist clinical decision-making by interpreting complex diagnostic data, such as MRI or CT scans. It increases diagnostic efficiency and reduces human error  in a high pressure clinical environment. Early detection of brain tumors is an important factor for a patient's survival. It allows the patient to get less invasive surgical interventions and more effective radiotherapy. If a diagnosis is missed or delayed, a patient can suffer severe neurological damage, loss of motor function and reduced life expectancy.

The “Alexnet moment” in the ImageNet competition in 2012 is widely regarded as a turning point which ignited the current AI revolution, especially in healthcare. A Convolutional Neural Network trained on GPUs was able to reduce error rate from 26% to 15% by extracting visual features automatically from raw pixels. These capabilities gave CNN models an edge over traditional methods that required “hand-crafted” feature engineering. Deep learning introduces multiple layers of abstraction to find patterns in intricate structures in large datasets to allow data processing (LeCun, Bengio and Hinton, 2015). In neuro-oncological context, deep learning has shown superiority at multi-class classification for brain tumors compared to standard machine learning (Sultan, Salem and Al-Atabany, 2019).

A clinically relevant AI tool must be able to differentiate between distinct pathological types, primarily Gliomas, Meningiomas, and Pituitary tumours.
Glioma: A type of tumour that starts in the glial (supportive) cells of the brain. They are often invasive and can be highly malignant.
Meningioma: A tumour that arises from the meninges (the membranes covering the brain and spinal cord). They are usually slow-growing and often benign.
Pituitary Tumour: Abnormal growths that develop in the pituitary gland at the base of the brain. These can interfere with hormone levels and affect vision.
1.2 Objective
The goal of this study is to investigate the application of deep learning in brain tumor classifications from MRI Scan images. A robust preprocessing pipeline is constructed to systematically evaluate the effects of the architectural improvements, data segmentation and transfer learning on classification accuracy across four tumor categories (Glioma, Meningioma, Pituitary, and No Tumour).

The study designs a progression from a standard Artificial Neural Network (ANN) to a custom Convolutional Neural Networks (CNNs) to evaluate the importance of spatial feature extraction in medical imaging. The CNN model is saved as a baseline model and compared with ANN and EfficientNetB0. EfficientNetB0 a pretrained CNN baseline model of the EfficientNet family. This model is designed to achieve high accuracy in image classification tasks. This model also ensures that the pipeline is highly efficient in terms of parameters and computational costs (Tan and Le, 2019).
Each model is benchmarked against baseline CNN by measuring the incremental gain from each improvement, such as regularization, Otsu segmentation, and transfer learning. Benchmark metrics include accuracy, macro F-1, macro precision and macro recall. The study also identifies three common machine learning challenges, overfitting, class imbalance and data leakage.
1.3 Dataset Overview
The Brain Tumor MRI Dataset was used from the kagglehub repository named "masoudnickparvar/brain-tumor-mri-dataset". The dataset consists of 7200 images which belong to four distinct classes, namely Glioma, Meningioma, Pituitary and No Tumour. Each class has 1800 images which denotes that the dataset is perfectly balanced (25% each) which shows equal representation. Of 1800 images for each category, 1400 are for training and 400 are for testing. The database has two folders, Testing and Training. Each of these folders has four folders labeled glioma, meningioma, notumor and pituitary which holds the respective images. Unlike most medical dataset, no resampling or weighting was required.

All images are resized to 224x224 pixels. Even though the MRI scans are greyscale, loaded as 3-channel RGB arrays as Keras/TensorFlow’s ImageDataGenerator loads images as RGB by default. EfficientNetB0 also expects 3-channel input since it was pretrained on ImageNet (which consists of colored photos).

Figure 1.3.2: Sample MRI image by class

The Glioma images can be identified by an irregular, infiltrating bright mass that spreads into surrounding brain tissue, making boundaries difficult to define. Meningioma tumor can be identified as a distinct bright region near the edge of the brain, often pressing against the skull. Pituitary tumours are identified as a small, well-defined bright spot in the central lower region of the brain. No tumor brains show a normal brain structure with no concentrated brain mass. Some scans contain variation in contrast and orientation.
2. Methods
2.1 Preprocessing
The Brain Tumor MRI Dataset from the kagglehub repository named "masoudnickparvar/brain-tumor-mri-dataset" was accessed using the kagglehub library using the dataset_download function. After downloading, the dataset is saved in a local google drive folder for persistence. A stratified split of 70/15/15 was done so that each class is represented proportionally in each split. To prevent data leakage, data augmentation was applied to training splits only.

The validation and test splits are only rescaled instead of augmentation. Class weights are computed from training labels only. This way, the information from test and validation splits are not leaked to training, thus making the evaluation process transparent. The augmentation techniques used in this data preparation are rotation_range=15, zoom_range=0.10, horizontal_flip, width/height shift of 0.1 for each. Zooming simulates different scan distances. As tumors can appear on either side of a brain, horizontal flips were used to take care of this edge case.The image pixel values are rescaled to [0, 1] and then divided by 255. The images were also resized to a resolution of 224x244 which is a standard resolution for pretrained networks.
2.2 Data Segmentation
Otsu thresholding technique is used to separate bright brain pixels from black background pixels. This helps to avoid manual thresholding and minimized within-class intensity variancy (Otsu, 1979). Morphological closing is used to fill small holes in the mask. Largest contour extraction to keep only the main brain region. The pixels from the skull edges and scan background can unnecessarily activate CNN filters. Masking out these regions forces the model to attend to the pixel of tumor bearing tissues only.

3.5 Challenges
Overfitting
In the baseline CNN experiment, at epoch 20, the training accuracy was 88.60% whereas the validation accuracy was 83.45%. The val_loss rose from 0.386 to 0.493 between epoch 16 and 20. This issue was addressed by applying a dropout value of 0.4 in the improved CNN experiments. This dropout value of 0.4 forward passes zeroes randomly to 40% of the neurons which prevents co-adaptation (Srivastava et al., 2014). BatchNormalization was also introduced to normalize activations. This also balances the gradients. EarlyStopping included stopping epochs if training didn’t improve after 7 epochs. Even though it did not trigger, it was a safety check. Data augmentation techniques like rotating, horizontal flipping, zoom and shift were introduced to skip training sample memorisation. 

The Improved CNN experiments still showed some gap between training and validation accuracy, but the gap was minimized compared to baseline CNN. EfficientNetB0 showed the smallest gap which suggests that pretrained features generalises better than custom CNN models.
Class Imbalance
No class imbalance issue was present as the dataset was perfectly balanced. This was confirmed using scikit-learn’s compute_class_weight function which showed that all the class weights came out exactly as 1.0. This was a methodological precaution to verify that the dataset was balanced. Meningioma class had the lowest per-class F1 (0.8822 in EfficientNetB0 ExpA) because of visual similarity as meningioma’s appearance overlaps with glioma. This denotes that it was not a quantitative but a visual issue.
Data Leakage
Data leakage in a dataset is the leakage of information from the training set to the test set. This causes the evaluation result to be over optimistic. Data augmentation was only applied to training datasets only. If the augmentations were done on validation data, it would inflate the validation accuracy. If class weights were computed from all 5600 images instead of 3920 training images, the label distribution from the unseen data would have shaped training. Data leakage would also occur if global normalisation statistics (mean/std) calculated from the full dataset instead of using separate ImageDataGenerator instances for training and validation dataset.

All these data leakage scenarios were prevented by the following:
Data Augmentation was only applied to training data splits only.
Class weights were computed from the training dataframe only.
Separate ImageDataGenerator instances were used for training and validation data splits.
Zero data overlap was verified between training.
