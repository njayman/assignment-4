## WEEK 11 PDF — Anomaly Detection and Fraud Detection

### Part 1: Introduction — What is Anomaly Detection?
  • Anomaly detection = identifying data points that deviate significantly from expected behaviour
  • Three anomaly types: Point anomaly (single outlier, e.g. unusually high transaction), Contextual anomaly (normal globally but anomalous in context, e.g. high spend at unusual time), Collective anomaly (group of related instances that are anomalous together)
  • Applications: fraud detection, network intrusion, industrial fault detection, medical diagnosis
  • Key challenge: anomalies are rare by definition — class imbalance is inherent

### Part 2: Statistical Methods
  • Z-score: measures how many std devs a point is from the mean; flag if |z| > 3 (assumes Gaussian)
  • IQR method: flag points outside [Q1 − 1.5×IQR, Q3 + 1.5×IQR]; robust to non-normal distributions
  • Mahalanobis distance: accounts for correlations between features; multivariate generalisation of Z-score
  • Limitations: statistical methods assume fixed distributions; struggle with high-dimensional or non-linear data

### Part 3: ML Methods
  • Isolation Forest (Liu et al., 2008): anomalies are easier to isolate — they require fewer splits in a random tree; scores inversely proportional to average path length; fast, scales to high dimensions
  • Local Outlier Factor (LOF, Breunig et al., 2000): compares local density of a point to its neighbours; score > 1 = lower density than neighbours = anomalous; good for density-varying datasets
  • One-Class SVM: learns a boundary around normal data in a high-dimensional kernel space; anything outside = anomaly; suitable when you only have normal training examples (unsupervised)
  • Decision: Isolation Forest for large tabular data; LOF for local-structure anomalies; One-Class SVM for kernel-based detection

### Part 4: Deep Learning Methods
  • Autoencoder: encoder compresses input to latent vector; decoder reconstructs; high reconstruction error = anomaly; trained on normal data only; limitation: reconstruction error threshold is manual
  • Variational Autoencoder (VAE): learns probabilistic latent distribution (mean + variance); anomaly score = reconstruction loss + KL divergence; better uncertainty quantification than plain AE
  • Deep SVDD (Ruff et al., 2018): learns a hypersphere in latent space enclosing normal data; anomaly score = distance from hypersphere centre; no reconstruction needed
  • Deep learning anomaly detection is especially powerful for image, text, or time-series where hand-crafted features are hard to design

### Part 5: Credit Card Fraud Case Study
  • Dataset: 284,807 transactions (Sept 2013, European cardholders); 492 fraud = 0.17% — extreme class imbalance
  • Features: V1–V28 (PCA-transformed for anonymity), Amount, Time; binary label: 0=normal, 1=fraud
  • Challenge: standard accuracy misleading — predicting all-normal gives 99.83% accuracy; must use precision/recall/F1/AUC-PR
  • Techniques applied: SMOTE oversampling on training set only (never on test), class_weight='balanced', threshold tuning
  • Model comparison: Logistic Regression, Random Forest, XGBoost, Autoencoder; ensemble of all gave best AUC-PR
  • Key lesson: for imbalanced fraud detection, recall for fraud class is safety-critical; false negatives (missed fraud) >> false positives (unnecessary block)
  • Business impact: even 1% improvement in fraud recall = millions saved at transaction scale

### Part 6: Takeaways — Anomaly Detection
  • No single best method: statistical (fast, interpretable), ML (scalable), DL (complex patterns)
  • Always use threshold tuning and precision-recall curves — not just accuracy
  • Data leakage is critical: SMOTE and normalisation stats must be computed on training set only
  • Ensemble anomaly detection (combine multiple detectors) typically outperforms any single model

## WEEK 12 PDF — Time Series Prediction

### Part 1: Foundations — What is a Time Series?
  • Time series = sequence of values indexed by time; exhibits trend, seasonality, cyclical patterns, irregular noise
  • Stationarity: a stationary series has constant mean, variance, and autocorrelation over time — required for ARIMA
  • ADF test (Augmented Dickey-Fuller): H0 = series has unit root (non-stationary); reject H0 (p < 0.05) → stationary
  • KPSS test: H0 = series is stationary; reject H0 → non-stationary; use both together for confirmation
  • Differencing: subtract t−1 from t to remove trend; seasonal differencing removes periodic pattern

### Part 2: Classical and ML Methods
  • Linear Regression with lag features: lag_1, lag_7, lag_14, rolling_mean_7 as regressors; simple and interpretable; 43% RMSLE improvement over naive baseline in Retail Sales case study
  • ARIMA(p,d,q): p=autoregressive lags, d=differencing order, q=moving average terms; Seasonal ARIMA adds (P,D,Q)_m for seasonal component; requires stationarity check first; 6% gain over LR in case study
  • XGBoost for time series: treated as supervised learning with lag features; RMSLE 0.45 (47% vs naive); top features: lag_1 (22.5%), lag_7 (15.2%), onpromotion (12.8%); production-ready, CPU-fast (2-5 min)
  • Key lesson: 80% of predictive power comes from temporal lag features — feature engineering > algorithm choice

### Part 3: Recurrent Neural Networks
  • RNN (1990s): h_t = activation(W_h · h_{t-1} + W_x · x_t + b); processes sequential data with temporal dependencies; limitation: vanishing gradient for long sequences
  • LSTM (Hochreiter & Schmidhuber, 1997): 3 gates (forget, input, output) + cell state; solves vanishing gradient; application: short-medium forecasts 1-30 days; limitation: slower than feedforward, limited multivariate support; RMSLE 0.42 (51% vs naive) in case study
  • GRU (Cho et al., 2014): 2 gates (reset, update) instead of 3; 30-40% faster training than LSTM; slightly less expressive; good for real-time/resource-constrained forecasting
  • BiLSTM (Schuster & Paliwal, 1997): processes sequence forward AND backward; better contextual understanding; cannot use for online forecasting (requires future data); doubles computational cost

### Part 4: Advanced DL Architectures
  • N-BEATS (Oreshkin et al., 2019) — M4 Competition Winner: pure MLP architecture outperforms RNNs; decomposes forecast into Trend + Seasonality + Residual stacks with backward/forward residual connections; y_t = Σ(seasonality) + Σ(trend) + Σ(residual); fast training (30-60 min), no GPU needed, highly interpretable; best for single variable, small data (<50K)
  • NHiTS (Challu et al., 2022): multi-rate hierarchical architecture; 3 levels (high/medium/low frequency) each with different context window sizes; interpolation connects predictions across levels; better than N-BEATS on long-horizon tasks (30-365 days)
  • PatchTST (Nie et al., 2022): divides time series into overlapping patches (tokens, 8-16 timesteps); applies Transformer encoder with multi-head attention; channel-independent (each variable treated separately); state-of-the-art accuracy for large multivariate datasets (100K+)
  • Model selection guide: small data → N-BEATS; long-horizon → NHiTS; multivariate/large → PatchTST; interpretability critical → N-BEATS/NHiTS; real-time CPU → LSTM or N-BEATS

### Part 5: Retail Sales Prediction Case Study
  • Dataset: Kaggle Store Sales Forecasting; 54,000 products × 54 stores × 1,681 days (2013-2017); 3M+ transactions; task: predict 16 days ahead for 33 stores
  • Feature engineering pipeline (50+ features): time-based lags (lag_1, lag_7, lag_14, lag_30, lag_365), rolling stats (rolling_mean_7, rolling_std_14), seasonal/calendar (day_of_week_sin/cos, is_holiday, is_payday), external (oil_price_lag, holiday_flag), store/product encodings
  • Baseline: Naive RMSLE 0.85, Linear Regression 0.72, ARIMA(2,1,2)×(1,1,1)_7 = 0.68
  • Advanced: XGBoost RMSLE 0.45, LSTM RMSLE 0.42; business impact: 45% forecast error reduction = $28MM annual savings, $55MM total
  • Key challenges: multiple seasonalities (weekly/monthly/yearly), zero-inflated data, irregular events (earthquakes, holidays)

### Part 6: Takeaways — Time Series
  • Feature Engineering > Algorithm: 50+ lag/rolling features drive 70% of gains
  • Always use TimeSeriesSplit (not random KFold) to prevent future data leakage
  • Check stationarity first (ADF/KPSS) before ARIMA; apply differencing if non-stationary
  • Deep hierarchy: LSTM < N-BEATS < NHiTS < PatchTST (accuracy, but also complexity)
  • Ensemble (XGBoost + N-BEATS + PatchTST) = competition gold standard
  • Common pitfalls: future leakage in CV, skipping stationarity, wrong metric (RMSE ignores cost asymmetry → prefer MASE), arbitrary lag selection without ACF/PACF analysis
