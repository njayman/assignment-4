3e. Challenges Encountered (~500 words)

Challenge 1 — Overfitting

Write a paragraph covering:
- Evidence: state the specific numbers — at epoch 20, training accuracy was 88.60% versus validation accuracy 83.45%. Also note that val_loss rose from
0.386 at epoch 16 to 0.493 at epoch 20.
- What this means: the model stopped learning general features and started memorising training-specific patterns and noise.
- How you addressed it — list each technique used in improved models:
  - Dropout(0.4) — each forward pass randomly zeros out 40% of neurons, preventing co-adaptation (cite Srivastava et al., 2014)
  - BatchNormalization — normalises activations between layers, acts as mild regularisation and stabilises gradients
  - EarlyStopping — was included as a callback (patience=7) though it did not trigger, it acts as a safety net
  - Data augmentation — rotation, flipping, zoom, and shift create artificial variety, reducing memorisation of exact training samples
- Was it resolved? Write honestly: the improved CNN still showed some gap between train and val accuracy, but it was reduced compared to the baseline.
EfficientNetB0 showed the smallest gap, suggesting pretrained features generalise better.

Challenge 3 — Data Leakage

Write a paragraph covering:
- Define it in your own words: data leakage occurs when information from the validation or test set influences the training process, making evaluation
results unrealistically optimistic.
- Give three concrete ways it could have happened in this project:
  a. Applying augmentation to validation images — the model would have been trained to classify already-augmented images, inflating reported val accuracy.
  b. Computing class weights from all 5,600 images instead of just the 3,920 training images — the label distribution of unseen data would have shaped
training.
  c. Using global normalisation statistics (mean/std) calculated from the full dataset rather than training data alone.
- How you prevented it: all three were addressed — augmentation only applied to train_datagen, class weights computed from train_df only, separate
ImageDataGenerator instances for train vs val/test (val/test used rescale only).
- Write 1 sentence confirming that zero file overlap was verified between train, val, and test splits (the notebook explicitly printed: "File overlap
across splits: 0").
